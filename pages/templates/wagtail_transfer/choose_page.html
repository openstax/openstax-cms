Importing content from another environment

Pulls a page or snippet into this site from the environment you pick below. Nothing changes on the source. Step-by-step walkthrough with screenshots (Scribe) →

  1. Publish your changes on the source first — drafts don't get exported.
  2. Pick the source environment, then find the page or snippet.
  3. For a brand-new page, pick a destination parent page.
  4. Review and import. Referenced images, documents, and snippets come across automatically.
  5. Check the result here before sharing a link to it.
  • To rebuild an entire environment (for example, refreshing dev or staging from production), don't import page by page — follow docs/refresh-from-prod-runbook.md instead.
  • Stuck or unsure? Ask in #openstax-dot-org before importing — imports overwrite content and can't be undone from this screen.
What actually gets imported
  • Draft edits never transfer. The export reads the source page's currently published content, not a pending draft revision — publish on the source first or you'll import stale content.
  • Published state carries over. A published source page is published here too, automatically — no extra "Publish" click. A source page that's never been published imports as a draft here.
  • Comes across: title, slug, SEO fields, body content, and anything it references — images, documents, collections, tags, and snippets.
  • Does not come across: owner, lock status, scheduled go-live/expiry dates, and revision history (a single fresh revision is created here instead).
Updating a page vs. replacing one at a different URL

The common case — re-importing a page you've already brought over: Wagtail Transfer remembers it, so the import updates it in place — same URL, same tree position — no matter what destination parent you pick above. Rebuild on staging, re-import to prod as many times as you like.

The rare case — a page built under a different slug (for example, staging an "FAQ" rebuild at /faq-staxify to later replace the live /faq): Wagtail Transfer has never linked the two, so importing it creates a brand-new page alongside the old one. A page created independently on both environments hits this same case — see docs/wagtail-transfer-page-sync.md. To swap the new one into the old URL:

  1. Import the new page here, into production.
  2. On the old live page, change the title (e.g. add "(old)") and slug (e.g. add "-old"), then save as a draft.
  3. Unpublish the old page. The URL is briefly down until the next step.
  4. On the newly imported page, set the title and slug back to the original values, then publish.

The home page is the one exception — don't import over it. Use "Promote to home page" on the new FlexPage instead.
