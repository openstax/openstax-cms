Importing content from another environment

This page pulls content into this site from the environment you select below. Nothing is changed on the source environment.

  1. Get the content ready on the source. Pages come across in their current state on the source environment, so finish and publish your changes there first.
  2. Select the source environment from the dropdown (for example, staging when you are promoting content to production).
  3. Find the page or snippet you want to import by browsing the source's page tree or snippet list.
  4. For a new page, choose where it should live on this site by picking the destination parent page.
  5. Review and import. Images, documents, and snippets that the content references are brought across automatically.
  6. Check the result. Open the imported page or snippet on this site and confirm it looks right before sharing links to it.

Good to know before you import:

  • If the item was imported before (or a snippet here shares its name/title with the source's), the import updates the existing item in place — any edits made on this site since the last import are replaced by the source version.
  • A page that was created separately on both environments is not recognized as the same page — importing it creates a duplicate. See docs/wagtail-transfer-page-sync.md in the CMS repository before importing a page that already exists here.
  • To rebuild an entire environment (for example, refreshing dev or staging from production), don't import page by page — follow docs/refresh-from-prod-runbook.md instead.
  • Stuck or unsure? Ask in the CMS support channel before importing — imports overwrite content and can't be undone from this screen.
