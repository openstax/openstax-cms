Experiments & Measurement

Run measurable website changes with PostHog feature flags & experiments. A/B test a change, ramp it with a percentage rollout, and read the result.

Where to go

  • PostHog project (create flags & experiments)
  • Framework overview
  • Change-and-measure runbook (step by step)

The short version

  1. Create a feature flag in PostHog (boolean for a rollout, multivariate for an A/B).
  2. Name it surface-first, kebab-case — e.g. nav-products-label, nav-k12-item.
  3. Pick a goal metric — an event already flowing (a nav click, book_engagement_clicked, a form submit).
  4. Ramp a rollout 10% → 25% → 100% in PostHog. No code change.
  5. Read lift & significance in PostHog's Experiments tab.
